Урок 2. Типы мобильных приложений
1. Нативные приложения
Это приложения, которые разработаны для конкретной мобильной операционной системы (ОС) — например, iOS или Android — 
с использованием языков и инструментов, рекомендованных вендором. 
Так, актуальный язык программирования для iOS — Swift (ранее был широко распространен Objective-C), для Android — Kotlin или Java.
Ключевые характеристики нативных мобильных приложений — высокая производительность и нативный интерфейс согласно гайдлайнам.
Преимущества
- Отличный пользовательский опыт (UI/UX)
- Доступ к аппаратным функциям. 
Приложение может сразу использовать камеру, геолокацию, микрофон
- Работа в автономном режиме (офлайн)
Недостатки
- Высокая стоимость и время разработки. 
- Обновления должны проходить проверку в магазинах приложений. 

2. Гибридные приложения
2.1. это нативное приложение, которое использует элемент WebView (компонент для отображения веб-контента)
2.2. это приложения, разработанные с использованием веб-технологий (HTML, CSS, JavaScript) и 
затем упакованные в нативный контейнер для мобильных платформ
Наиболее популярные фреймворки, которые можно отнести к этому подходу — React Native, Flutter, Kotlin Multiplatform (KMP).
Преимущества
- Общая кодовая база для бизнес-логики, но подход к UI может различаться в зависимости от фреймворка.
- Производительность близка к нативной, высокая скорость разработки
Недостатки
- Риск снижения производительности из-за выполнения кода через фреймворк
- Ограниченный доступ к нативным функциям
- Доступ к новым API устройства может появляться с задержкой
- Баги фреймворка 

3. Progressive Web Apps (PWA)
это не приложения в чистом виде, а усовершенствованные веб-сайты.
Доступ к ним может осуществляться через браузер или путем установки на устройство пользователя.
PWA могут работать в автономном режиме или при нестабильном подключении к интернету благодаря использованию кэша,
могут отправлять push-уведомления, получать доступ к функциям устройства — например, камере или геолокации,
не зависят от платформы.
Самый яркий пример использования PWA — банковские приложения.
Чтобы сайт стал PWA, нужно реализовать два основных компонента: Web App Manifest и Service Workers. 
Web App Manifest - SON-файл, который сообщает браузеру, как приложение должно выглядеть после установки на устройство.
Service Worker — это скрипт, который работает в фоновом режиме отдельно от веб-страницы. Необходим для пушей
